Deployment to Cloudflare Pages

Ari 1.x only support to deploy to Cloudflare Pages. The deployment process is very great simple and easy as described below.

  1. You have your own local development environment, modify Ari as you wish. Tested locally.

  2. Create a GitHub repository (private repository) and push Ari to your GitHub repository.

  3. Connect your GitHub account to Cloudflare Pages.

And everything would be executed automatically. Every time you update your GitHub, your Ari would be deployed to Cloudflare Pages automatically.

info

Ari could be ported to run on any serverless platform (thanks to Remix Web Framework) such as:

  • AWS
  • Cloudflare worker
  • FlyIo
  • Netlify
  • Vercel
  • Any Express server

but for the very first version of Ari (1.x), we only support to deploy to Cloudflare Pages. The reason is that Ari (1.x) focus on standalone spirit and effordable spirit. You would have a powerful Serverless Personal Website for FREE when deploy to Cloudflare Pages
